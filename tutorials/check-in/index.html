How to check-in attendees

Please note that the check-in app needs an active internet connection in order to validate the tickets.

Configure the account

  • Install the Android application on your Smartphone from the Google Play Store

  • Create Check-in operator account

    Go to the admin section of your alf.io instance, create a new user for your organization, and grant him/her the "Check-in Operator" role

  • Password generation

    Once the user has been created, alf.io will prompt the auto-generated password for that user. Keep that window opened, you will use the generated password right away.

Configure the Android App

  • Scan the password within the application

    Click on the "+" button and select "Scan QR-Code" from the menu. Then scan the generated password.

  • Select the event (optional)

    Each organization can have more than one active event. If that's your case, you will be asked to select one event from the list. You can add other events by scanning again the generated password.

Scan attendees' badge

  • Select the event from the list

    Once you have configured the event, select the corresponding profile from the list.

  • Init check-in

    Click on "Scan Tickets" in order to start the check-in process.

Possible scan results

Vibration feedback: once means "successful", three times: "error / action required"

  • Check-in successful *

    Everything's OK, the Attendee can enjoy the event

  • Attendee must pay the ticket upon arrival ***

    An outstanding payment has been found. Once it has been cleared out, click on "Confirm" to perform check-in

  • Ticket already checked in ***

    A ticket can be checked-in only once. This error doesn't necessarily mean that the attendee is trying to cheat.
    You have to verify if his/her ticket is part of a group reservation, maybe he/she is just showing the wrong one.

  • Ticket not found ***

    The system is unable to find a match for the scanned ticket, could be one of the following:

    1. You have selected the wrong event from the list
    2. The Attendee has modified the ticket (he/she is still the assignee) and is showing the old one
    3. The Attendee has reassigned the ticket to someone else and is trying to access the event anyway

I don't have an android device. What can I do?

Currently we don't have enough resources to develop a native application for iOS. Any help (sponsoring and/or code contribution) would be highly appreciated.

In any case, you can still use the check-in functionality of the admin, which allows you to search attendees and do a manual check-in. Access it either from the dashboard or from event detail.
